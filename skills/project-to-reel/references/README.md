# Reference files you supply

`project-to-reel` is a framework. Its quality comes from these reference files, which encode
YOUR content strategy. They are intentionally not shipped here (they are yours to write, and
mine are my own brand IP). Create the ones you want; the skill reads them on demand.

| File | What it holds | Read when you need to... |
|------|---------------|--------------------------|
| `strategy.md` | Your niche, content franchises (the fixed set of video types you make), claim discipline, production grammar. The source of truth. | Pick the franchise, check claims |
| `hook-library.md` | Your hook patterns with examples (pain-fix, contrarian, time-money, demo-first, personal-stake, ...). | Generate hooks |
| `claim-blocklist.md` | Claims you will never make, and softening rules. | Block a risky claim |
| `algorithm-evidence.md` | Sourced platform behaviors you are willing to cite (with links). | Cite a platform rule |
| `psychology.md` | The retention/attention principles you design against. | Justify a script or edit decision |
| `creator-tactics.md` | Creators whose style you match, with reference video URLs. | Match a style |
| `editing-grammar.md` | Your cut rules, caption style, pacing. | Apply editing rules |
| `tool-stack.md` | Your hardware/software recommendations. | Recommend gear |

Optional: a `prompts/` folder with one prompt file per pipeline stage if you want the
per-stage instructions separated from `SKILL.md`, and a `templates/` folder with your
`brief.html` / `brief.md` / editor-handoff templates.

Start with `strategy.md`. Without it, the skill produces a competent skeleton, not your voice.
